# pangolin-eval v0.1.0

`pangolin-eval` is a local-first CLI and Python package for measuring LLM, RAG,
and agent workloads across cost, latency, quality, and reliability.

## Highlights

- Compare models and prompts with no hosted service required.
- Generate JSON, Markdown, and optional static HTML reports.
- Keep sensitive response text out of saved artifacts with metadata-only mode.
- Enforce cost, latency, quality, and reliability gates in CI.
- Evaluate synthetic RAG tasks for coverage, faithfulness, citation, and context waste.
- Summarize local agent/workflow traces with TraceCards.
- Export comparison reports and TraceCards as OTel-style spans.
- Use provider templates for OpenAI-compatible APIs, LiteLLM, Ollama, and vLLM.

## Package

- Python: 3.9+
- Runtime dependencies: none
- CLI entrypoint: `pangolin-eval`

## Verification

Release readiness should include:

```bash
PYTHONPATH=src python3 -m unittest discover -s tests
PYTHONPATH=src python3 -m pangolin_eval.cli validate --config examples/simple_model_compare/config.json
PYTHONPATH=src python3 -m pangolin_eval.cli run --config examples/simple_model_compare/config.json --out reports/simple_model_compare --content-mode metadata-only --html
PYTHONPATH=src python3 -m pangolin_eval.cli rag --config examples/rag_eval/config.json --out reports/rag_eval --content-mode metadata-only --html
PYTHONPATH=src python3 -m pangolin_eval.cli trace --input examples/agent_trace/trace_events.json --out reports/agent_trace --html
```

GitHub Actions should pass on Python 3.9, 3.10, 3.11, and 3.12.
