# pangolin-eval v0.2.0

`v0.2.0` is a post-release polish release focused on compatibility, release
hygiene, and clearer public examples.

## Highlights

- Added a public compatibility policy for CLI, config, report, RAG, TraceCard,
  pricing, and export contracts.
- Added migration notes for moving from `v0.1.0` to `v0.2.0`.
- Added a CI-gate example config for budget, latency, quality, and reliability
  thresholds.
- Added version-consistency regression coverage across package metadata.
- Updated release docs for the `v0.2.0` tag flow.

## Compatibility

No breaking changes are expected from `v0.1.0`.

Report consumers should continue branching on `schema_version` rather than
package version.

## Verification

Release readiness should include:

```bash
PYTHONPATH=src python3 -m unittest discover -s tests
find schemas examples -name '*.json' -print0 | xargs -0 -n1 python3 -m json.tool >/dev/null
PYTHONPATH=src python3 -m pangolin_eval.cli validate --config examples/ci_gate/config.json
PYTHONPATH=src python3 -m pangolin_eval.cli run --config examples/ci_gate/config.json --out reports/ci_gate --content-mode metadata-only --html
```

GitHub Actions should pass on Python 3.9, 3.10, 3.11, and 3.12.
